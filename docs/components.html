The Jarvis component reference

Every building block that ships with the Jarvis theme, explained in plain language — the 16 components you drag into a page, plus the two small modules that make them work. Written for site builders who are new to Single Directory Components but comfortable in Drupal.

Start hereWhat are these components?

Jarvis is built out of Single Directory Components — SDCs for short. If the term is new: an SDC is a small, self-contained piece of a page (a hero banner, a card, a two-column row) that lives in its own folder inside the theme. Each folder holds three things — a .component.yml file that lists the settings the component accepts, a .twig template that renders the HTML, and its own CSS. Drupal core discovers them automatically. Nothing to install.

The practical payoff: an editor can drop a "Hero" onto a page and fill in a heading, a background image, and a button — without touching code — and always get the same tidy, accessible markup. The settings an editor fills in are called props (short for properties). Some components also have slots, which are drop-zones where you nest other components; that's how the layout pieces hold your content.

You'll meet Jarvis's components in two places. Drupal Canvas is the visual, drag-and-drop page builder — that's the primary home for these. Layout Builder is Drupal's older layout tool; the two custom modules at the end of this page let the same components work there too.

How to read this page

Start with Foundations. Almost every component shares the same spacing controls, the same background-and-contrast behaviour, and the same approach to images. Learning those three patterns once means every component table afterward is short — it only lists what makes that component different.

FoundationsThe shared spacing controls

Fifteen of the sixteen components (everything except Button) expose the same set of spacing props. Rather than repeat them in every table below, here they are once. When a component's badge says "＋ standard spacing controls," it means this set.

Under the hood they all funnel through one shared Twig helper, jarvis-spacing.html.twig, which turns your choices into utility classes (jp-p-*, jp-m-*, jp-vpad) defined in the theme's overrides.css. One source of truth — change spacing behaviour there and every component follows.

{#
/**
 * @file
 * Shared spacing utility-class builder for the Jarvis SDCs.
 *
 * Maps the padding/margin/vertical_padding/no_padding props every component
 * repeats onto the jp-* utility classes in css/overrides.css. Imported as
 * `{% import '@jarvis/jarvis-spacing.html.twig' as jarvis %}` — call sites
 * pass |default()-guarded props so undefined props are safe under
 * strict_variables. One source of truth: change spacing behaviour here.
 */
#}
{% macro classes(padding, padding_size, margin, margin_size, vertical_padding, no_padding) -%}
  {{- [
    padding != 'none' ? 'jp-p-' ~ padding ~ '-' ~ padding_size : '',
    margin != 'none' ? 'jp-m-' ~ margin ~ '-' ~ margin_size : '',
    vertical_padding ? 'jp-vpad' : '',
    no_padding ? 'jp-nopad' : '',
  ]|join(' ')|trim -}}
{%- endmacro %}

FoundationsBackgrounds & the contrast guard

The four layout containers — 1 Column, 2 Columns, 3 Columns, and Card Full Image — share a background system. Rather than repeat it, here's how it works.

You can give a container a solid colour or a background image. If you pick a colour and an image, the colour wins — the image is hidden but not deleted, so you can flip back by choosing "None" again. Over an image, an adjustable dark (or light) overlay keeps text readable.

The clever part is the contrast guard. You can force a text colour on a whole section with text_color, but Jarvis won't let you pick an unreadable combination. It checks your choice against the section's own background using WCAG AA (the accessibility contrast standard). If it passes, your colour is used; if it fails, Jarvis quietly falls back to a colour it knows is legible. Over a photo — where brightness is unknowable — it honours your pick and leans on the overlay to carry readability.

FoundationsHow images work in Canvas

Several components accept an image (Hero, Card, Image, Person, Stat…). In Canvas, an image prop arrives as a small bundle of values — a ready-to-use src URL, an alt text, and often width/height. You pick the file in the editor; Jarvis handles the markup, lazy-loading, and sizing.

One wrinkle worth knowing about, because it explains a couple of otherwise-mysterious props. Drupal image styles (named crops like "Thumbnail" or "Wide") normally need a file's internal address (public://…), but Canvas only hands components a finished public URL. So Jarvis ships a small helper — the jarvis_image_style Twig filter, in the Jarvis Canvas module — that looks the file back up by its name and applies the style anyway. That's why the Image component has an image_style picker, and why Person has a hidden image_uri prop feeding its portrait crop.

 * Count-up animation for stat cards. Animates [data-count-up] from 0 to its
 * number when scrolled into view. Preserves prefix/suffix (e.g. "1,200+").
 */
((Drupal, once) => {
  // ponytail: parse "$1,200+" -> {prefix:"$", value:1200, suffix:"+", commas:true}
  const parse = (raw) => {
    const m = raw.match(/^(\D*)([\d,]*\.?\d+)(\D*)$/);
    if (!m) return null;
    return {
      prefix: m[1],
      value: parseFloat(m[2].replace(/,/g, '')),
      suffix: m[3],
      commas: m[2].includes(','),
      decimals: (m[2].split('.')[1] || '').length,
    };
  };

  const render = (p, n) => {
    let s = n.toFixed(p.decimals);
    if (p.commas) s = Number(s).toLocaleString(undefined, { minimumFractionDigits: p.decimals });
    return p.prefix + s + p.suffix;
  };

  const run = (el) => {
    const p = parse(el.textContent.trim());
    if (!p) return;
    const dur = 1600;
    let start = null;
    const step = (ts) => {
      if (start === null) start = ts;
      const t = Math.min((ts - start) / dur, 1);
      const eased = 1 - Math.pow(1 - t, 3); // easeOutCubic
      el.textContent = render(p, p.value * eased);
      if (t < 1) requestAnimationFrame(step);
    };
    requestAnimationFrame(step);
  };

  Drupal.behaviors.jarvisStatCountUp = {
    attach(context) {
      once('jarvis-count-up', '[data-count-up]', context).forEach((el) => {
        if (!('IntersectionObserver' in window)) return run(el); // ponytail: old browsers just see final value
        const io = new IntersectionObserver((entries) => {
          entries.forEach((e) => {
            if (e.isIntersecting) {
              run(el);
              io.disconnect();
            }
          });
        }, { threshold: 0.4 });
        io.observe(el);
      });
    },
  };
})(Drupal, once);

<?php

/**
 * @file
 * Renders Jarvis content block types through their matching SDCs.
 *
 * A single preprocess hook covers both Layout Builder inline blocks and
 * Block-layout placed content blocks: both return the block_content entity
 * view build, so #block_content is present on $variables['content'].
 */

declare(strict_types=1);

use Drupal\block_content\BlockContentInterface;

/**
 * Maps block_content bundle => SDC component id.
 */
function _jarvis_blocks_map(): array {
  return [
    'hero' => 'jarvis:hero',
    'card' => 'jarvis:card',
    'video' => 'jarvis:video',
    'image' => 'jarvis:image',
    'text' => 'jarvis:text',
  ];
}

/**
 * Implements hook_preprocess_HOOK() for block.
 */
function jarvis_blocks_preprocess_block(array &$variables): void {
  $content = $variables['content'] ?? [];
  $bc = $content['#block_content'] ?? NULL;
  if (!$bc instanceof BlockContentInterface) {
    return;
  }
  $map = _jarvis_blocks_map();
  $bundle = $bc->bundle();
  if (empty($map[$bundle])) {
    return;
  }

  $props = _jarvis_blocks_props($bundle, $bc);

  // Replace the field-rendered body with the SDC render element. Keep the
  // original cache metadata so block/entity cache tags still bubble up.
  $component = [
    '#type' => 'component',
    '#component' => $map[$bundle],
    '#props' => $props,
  ];
  if (!empty($content['#cache'])) {
    $component['#cache'] = $content['#cache'];
  }
  $variables['content'] = $component;
}

/**
 * Builds SDC props from a block_content entity's fields.
 */
function _jarvis_blocks_props(string $bundle, BlockContentInterface $bc): array {
  $has = static fn (string $f): bool => $bc->hasField($f) && !$bc->get($f)->isEmpty();
  $str = static fn (string $f): ?string => $has($f) ? (string) $bc->get($f)->value : NULL;
  $bool = static fn (string $f): bool => $has($f) ? (bool) $bc->get($f)->value : FALSE;
  $int = static fn (string $f): ?int => $has($f) ? (int) $bc->get($f)->value : NULL;

  $html = static function (string $f) use ($has, $bc): ?string {
    if (!$has($f)) {
      return NULL;
    }
    // Canvas's TextLongItemOverride drops the `processed` computed property,
    // so run the text through check_markup() directly.
    $item = $bc->get($f)->first();
    return (string) check_markup($item->value, $item->format ?: NULL);
  };

  $image = static function (string $f) use ($has, $bc): ?array {
    if (!$has($f)) {
      return NULL;
    }
    $item = $bc->get($f)->first();
    $file = $item->entity;
    if (!$file) {
      return NULL;
    }
    $url = \Drupal::service('file_url_generator')->generateAbsoluteString($file->getFileUri());
    $out = ['src' => $url, 'alt' => (string) ($item->alt ?? '')];
    if (!empty($item->width)) {
      $out['width'] = (int) $item->width;
    }
    if (!empty($item->height)) {
      $out['height'] = (int) $item->height;
    }
    return $out;
  };

  $link = static function (string $f) use ($has, $bc): array {
    if (!$has($f)) {
      return [NULL, NULL];
    }
    $item = $bc->get($f)->first();
    try {
      $url = $item->getUrl()->toString();
    }
    catch (\Throwable $e) {
      $url = $item->uri;
    }
    return [$url, $item->title ?: NULL];
  };

  switch ($bundle) {
    case 'hero':
      [$cta_href, $cta_text] = $link('field_cta');
      return array_filter([
        'heading' => $str('field_heading'),
        'subheading' => $str('field_subheading'),
        'cta_text' => $cta_text,
        'cta_href' => $cta_href,
        'image' => $image('field_media_image'),
        'overlay' => $int('field_overlay') ?? 40,
        'align' => $str('field_align') ?? 'center',
      ], static fn ($v) => $v !== NULL);

    case 'card':
      [$link_href, $link_text] = $link('field_cta');
      return array_filter([
        'variant' => $str('field_variant') ?? 'image-top',
        'image' => $image('field_media_image'),
        'title' => $str('field_title'),
        'body' => $html('field_body'),
        'link_text' => $link_text,
        'link_href' => $link_href,
      ], static fn ($v) => $v !== NULL);

    case 'video':
      [$video_url] = $link('field_video_url');
      return array_filter([
        'video_url' => $video_url,
        'aspect' => $str('field_aspect') ?? '16x9',
        'full_width' => $bool('field_full_width'),
        'autoplay' => $bool('field_autoplay'),
        'muted' => $bool('field_muted'),
      ], static fn ($v) => $v !== NULL);

    case 'image':
      return array_filter([
        'image' => $image('field_media_image'),
        'caption' => $str('field_caption'),
        'full_width' => $bool('field_full_width'),
      ], static fn ($v) => $v !== NULL);

    case 'text':
      return array_filter([
        'body' => $html('field_body'),
        'width' => $str('field_width') ?? 'normal',
        'align' => $str('field_align') ?? 'start',
      ], static fn ($v) => $v !== NULL);
  }

  return [];
}

Custom modulesJarvis Canvas

jarvis_canvas is the glue between the Jarvis components and Drupal Canvas. It patches over a few rough edges in Canvas so the theme behaves the way editors expect. It does four things.

1 · Unlocks a real rich-text editor for the WYSIWYG component

Canvas normally forces every rich-text prop into one locked, stripped-down editor. This module watches for the x-jarvis-html-format marker (present only on the WYSIWYG component's body) and, for that prop alone, swaps in the site's real Basic HTML and Full HTML formats — giving editors the full CKEditor toolbar. Canvas offers no per-component setting for this, so the marker-on-the-prop trick is how the module scopes it to a single component.

2 · Restores the "Create with Canvas" button on Drupal 11.4

Drupal 11.4 reorganised the "Manage display" screen, and in doing so Canvas's own "Create with Canvas" call-to-action stopped appearing — meaning new content types couldn't get a Canvas template through the interface. This module re-adds that button on the new screen, but only when it's genuinely needed (the content type has no Canvas template yet, and the user is allowed to create one). It's a temporary workaround for Canvas issue #3498525.

3 · Gives every new content type a Canvas template automatically

Because of that same Canvas gap, creating a content type didn't reliably produce a Canvas template to design against. So whenever you create a new content type, this module automatically creates an empty Full content Canvas template for it — instantly ready to edit at /canvas/template/node/<type>/full. And when a content type is deleted, it tidily removes that template again (an empty template has no automatic link back to the type, so Drupal wouldn't clean it up on its own).

4 · The jarvis_image_style Twig filter

This is the helper mentioned back in Images in Canvas. Canvas hands components a finished image URL, but Drupal image styles need the file's internal address — so the filter looks the file up by its name and builds the styled crop from there. It's what makes the Image component's style picker actually work. (Like Canvas's own equivalent, it matches files by filename, which is fine unless two managed files somehow share the exact same name.)

/**
 * Implements hook_canvas_storable_prop_shape_alter().
 *
 * Canvas hardwires every block rich-text prop to its locked canvas_html_block
 * format. Swap in the site's Basic/Full HTML formats so editors get a real
 * text-format picker — but ONLY for props that explicitly opt in with the

 * shape normalization (see PropShape::normalizePropSchema), so it reaches this

  $schema = $storable_prop_shape->shape->schema;
  if (($schema['contentMediaType'] ?? NULL) === 'text/html'
    && !empty($schema['x-jarvis-html-format'])) {
    $storable_prop_shape->fieldInstanceSettings['allowed_formats'] = ['basic_html', 'full_html'];

  if (\Drupal::isConfigSyncing()) {

  if (\Drupal::isConfigSyncing()) {
    return;

    ->getStorage('content_template')
    ->load('node.' . $type->id() . '.full');
  if ($template) {

  }

<?php

declare(strict_types=1);

namespace Drupal\jarvis_canvas;

use Drupal\Core\Entity\EntityTypeManagerInterface;
use Drupal\Core\File\FileUrlGeneratorInterface;
use Drupal\file\FileInterface;
use Drupal\image\Entity\ImageStyle;
use Drupal\image\ImageStyleInterface;
use Twig\Extension\AbstractExtension;
use Twig\TwigFilter;

/**
 * Twig filter to apply an image style to a Canvas image src URL.
 *
 * The Jarvis image SDC receives its image as a resolved public URL
 * (canvas.module/image → src built with File::createFileUrl()). Image styles,
 * however, need the original stream-wrapper URI (public://…). Canvas never
 * populates a stream-wrapper prop on a plain component insert, so the built-in
 * |image_style filter has nothing to work with.
 *
 * This filter bridges the gap the same way Canvas's own ImageAndStyleAdapter
 * does: look the file up by basename, then build the styled URL from its URI.
 *
 * ponytail: basename lookup, exactly like ImageAndStyleAdapter. Collides only
 * if two managed files share a filename; upgrade to a fid-carrying prop if that
 * ever bites.
 */
final class JarvisCanvasTwigExtension extends AbstractExtension {

  public function __construct(
    private readonly EntityTypeManagerInterface $entityTypeManager,
    private readonly FileUrlGeneratorInterface $fileUrlGenerator,
  ) {}

  /**
   * {@inheritdoc}
   */
  public function getFilters(): array {
    return [
      new TwigFilter('jarvis_image_style', [$this, 'imageStyle']),
    ];
  }

  /**
   * Returns the styled, root-relative URL for an image, or NULL to fall back.
   *
   * @param string|null $src
   *   The image src as delivered to the component (a public file URL).
   * @param string $style
   *   The image style machine name.
   */
  public function imageStyle(?string $src, string $style): ?string {
    if (!$src) {
      return NULL;
    }
    $image_style = ImageStyle::load($style);
    if (!$image_style instanceof ImageStyleInterface) {
      return NULL;
    }
    // image.src maps to src_with_alternate_widths — a URL whose query string
    // ends in a different filename (…jarvis-banner.png.avif?itok=…). Strip the
    // query before basename or the file lookup matches nothing.
    $path = (string) parse_url($src, PHP_URL_PATH);
    $files = $this->entityTypeManager->getStorage('file')
      ->loadByProperties(['filename' => urldecode(basename($path))]);
    $file = reset($files);
    if (!$file instanceof FileInterface) {
      return NULL;
    }
    $uri = (string) $file->getFileUri();
    if (!$image_style->supportsUri($uri)) {
      return NULL;
    }
    return $this->fileUrlGenerator->transformRelative($image_style->buildUrl($uri));
  }

}
